# ИИ-обзвонщик с фиксацией лидов в Telegram

Робот обзванивает базу клиентов, проигрывает **вашу голосовую запись**, и если
клиент в конце нажимает **«1»** («…если вам интересно, нажмите 1») — его номер
сразу прилетает в **Telegram** с пометкой «🔥 ЛИД».

Реализация на **Voximplant** (облачная телефония): сам звонок, проигрывание
записи и сбор нажатия выполняются в облаке — свой сервер держать **не нужно**.
Python-скрипт только запускает обзвон из CSV.

```
┌──────────────┐   StartScenarios    ┌──────────────────┐   звонок    ┌────────┐
│ campaign.py  │ ──────────────────▶ │ Voximplant облако │ ─────────▶ │ Клиент │
│ (ваш ПК)     │   номер из CSV      │  scenario.js      │  проигрыш  └────────┘
└──────────────┘                     │  + ловит DTMF "1" │      запись     │
                                     └──────────────────┘                  │ нажал 1
                                              │ при нажатии "1"             │
                                              ▼                            ◀┘
                                       ┌──────────────┐
                                       │ Telegram бот │  «🔥 ЛИД: +7…»
                                       └──────────────┘
```

## Состав проекта

| Файл | Назначение |
|------|-----------|
| `voxengine/scenario.js` | Облачный сценарий: звонок → запись → сбор «1» → Telegram |
| `campaign.py` | Запуск обзвона из CSV через Voximplant Management API |
| `data/clients.example.csv` | Пример базы номеров (скопируйте в `data/clients.csv`) |
| `.env.example` | Шаблон настроек (скопируйте в `.env` и заполните) |
| `leads/campaign_log.csv` | Лог запусков звонков (создаётся автоматически) |

## Пошаговая настройка

### 1. Telegram-бот
1. В Telegram напишите **@BotFather** → `/newbot` → получите **токен**.
2. Узнайте свой **chat_id**: напишите боту **@userinfobot**, он пришлёт ваш id.
   (Для группы — добавьте бота в группу и возьмите id группы.)
3. **Важно:** один раз напишите своему боту любое сообщение (`/start`), иначе
   он не сможет вам писать.

### 2. Voximplant
1. Зарегистрируйтесь на <https://voximplant.com> (есть бесплатный тестовый
   кредит для проверки).
2. **Купите номер** (Caller ID), от которого будут идти звонки:
   *Numbers → Buy*.
3. Создайте **Application** (приложение): *Applications → Create*.
4. Озвучка сообщения — **два режима** (выбирается в `.env` / в `CONFIG` сценария):
   - **TTS (для теста):** ничего хостить не нужно, робот проговорит текст из
     `MESSAGE_TEXT`. Удобно для обкатки.
   - **mp3 (боевой, свой голос):** положите файл на публичный URL и пропишите
     его в `AUDIO_URL`, а `MESSAGE_TEXT` оставьте пустым.
   > В конце сообщения обязательно фраза «…если вам интересно, нажмите 1».
5. Создайте **Scenario**: *ваше приложение → Scenarios → Create*, вставьте
   содержимое `voxengine/scenario.js`.
   - Либо впишите токены прямо в `CONFIG` вверху файла,
   - либо оставьте плейсхолдеры и передавайте значения через `.env`
     (Python сам прокинет их в сценарий через `customData` — удобнее и
     безопаснее, токены не лежат в облаке).
6. Создайте **Rule** (правило): *ваше приложение → Routing/Rules*, привяжите
   к сценарию. Запомните **Rule ID** — он понадобится в `.env`.
7. Возьмите **Account ID** и **API key**: *Settings → API access*.

### 3. Локальная настройка
```bash
# Python 3.9+
python -m venv .venv && source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt

cp .env.example .env          # заполните значениями из шагов 1–2
cp data/clients.example.csv data/clients.csv   # вставьте свои номера
```

Заполните `.env`:
```
VOX_ACCOUNT_ID=...
VOX_API_KEY=...
VOX_RULE_ID=...
VOX_CALLER_ID=+7...
AUDIO_URL=https://.../voice.mp3
TELEGRAM_BOT_TOKEN=...
TELEGRAM_CHAT_ID=...
```

### 4. Проверка и запуск
```bash
python campaign.py --dry-run     # покажет, какие номера будут обзвонены (без звонков)
python campaign.py               # обзвон data/clients.csv
python campaign.py --file base2.csv   # другой файл базы
```

Во время обзвона лиды (нажавшие «1») приходят в Telegram сразу. Лог запусков —
в `leads/campaign_log.csv`.

## Формат базы (`data/clients.csv`)
Любой из вариантов подходит (номера нормализуются автоматически):
```
phone
+7 900 123-45-67
89001234568
79001234569
```
Можно без заголовка — тогда номер берётся из первого столбца.

## Частые вопросы

**Где хранятся лиды?** Основной канал — Telegram (мгновенно). Дополнительно
все *запуски* звонков логируются в `leads/campaign_log.csv`. Если нужен
отдельный список именно «нажавших 1» в файле — это делается небольшим
веб-хуком; скажите, добавлю.

**Сколько стоит?** Софт бесплатный. Платите только Voximplant за минуты
разговора (поминутно) и за аренду номера. Тестовый кредит позволяет проверить
без затрат.

**Можно несколько звонков одновременно?** Да, Voximplant это позволяет. Сейчас
для аккуратности стоит последовательный обзвон с паузой `DELAY_BETWEEN_CALLS`.
Параллельный режим добавлю по запросу.

**API не совпадает с доками?** Имена методов VoxEngine иногда меняются. Если
сценарий ругается — сверьтесь с актуальной справкой Voximplant
(<https://voximplant.com/docs/references/voxengine>); логика остаётся прежней.

## ⚠️ Юридическое
Массовый автообзвон с рекламой в РФ (ФЗ-38 «О рекламе», ст. 18) требует
**предварительного согласия абонента**. Обзвон «холодной» базы без согласия —
нарушение. Ответственность за легальность базы и согласий — на стороне
владельца базы.
