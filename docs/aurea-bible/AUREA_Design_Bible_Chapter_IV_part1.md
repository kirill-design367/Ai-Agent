# ГЛАВА IV. UI SYSTEM

*Полноценная дизайн-система. Материальная база для всех сцен Главы III.*

---

## 0. Тезис главы

Главы I–III дали философию, движение и драматургию. Но сцену нельзя построить из абстракций — нужны **конкретные материалы**: точные размеры шрифтов, шаги отступов, токены теней, состояния кнопок, правила сеток. Эта глава — **инструментарий**. Она превращает «дорого и воздушно» из ощущения в **числа и токены**, которыми можно собрать сайт.

> Философия без системы — болтовня. Система без философии — шаблон. AUREA — это система, **подчинённая** философии.

Важнейшее предупреждение, которое держит в голове каждый, кто читает эту главу: **токены — это не разрешение на единообразие**. Дизайн-система AUREA задаёт *палитру возможностей*, а не *обязательную раскладку*. Из одних и тех же токенов одна сцена собирается воздушной и тихой, другая — плотной и дерзкой (Гл. III, 1.5; Гл. VI). Система — это ноты, а не песня. Дизайнер, который из системы делает одинаковые экраны, — нарушает её дух, даже соблюдая букву.

## 1. Базовая единица и пространственная шкала

### 1.1. Базовая единица: 8px (с под-шагом 4px)

Вся пространственная система строится на сетке **8px**. Это даёт скрытую математическую гармонию: все отступы, размеры, выравнивания кратны 8 (или 4 для тонких настроек). Глаз не видит сетку, но чувствует порядок.

- **Базовый шаг:** 8px
- **Под-шаг (для тонкой типографики, иконок, бордеров):** 4px
- Всё остальное — кратно 8: 8, 16, 24, 32, 40, 48, 64, 80, 96, 128, 160, 192, 256…

### 1.2. Пространственная шкала (spacing tokens)

```
--space-3xs: 4px     микро-зазоры, тонкая типографика
--space-2xs: 8px     внутри плотных элементов
--space-xs:  12px    между связанными мелкими элементами
--space-sm:  16px    стандартный мелкий отступ
--space-md:  24px    между элементами группы
--space-lg:  32px    между группами
--space-xl:  48px    между подсекциями
--space-2xl: 64px    отступы внутри сцены
--space-3xl: 96px    между смысловыми зонами
--space-4xl: 128px   воздух сцены
--space-5xl: 160px   крупный межсценовый воздух
--space-6xl: 192px   премиальный макро-воздух
--space-7xl: 256px+  «дышащие» hero/финал зоны
```

### 1.3. Главное правило отступов: НЕРАВНОМЕРНОСТЬ

Прямо из Главы I (8.3, п.2). **Запрещены одинаковые вертикальные отступы между всеми сценами** (это «бутстрап-ритм», Гл. VI). Отступы варьируются:
- между **связанными** элементами — малые (`xs`–`md`);
- между **смысловыми группами** — большие (`xl`–`3xl`);
- между **сценами** — крупные и **разные** (`4xl`–`7xl`), задающие индивидуальный ритм каждой сцене.

Закон близости: расстояние кодирует связь. Равные отступы убивают иерархию и читаются как механика, дёшево.

### 1.4. Принцип «воздух пропорционален важности»

Чем важнее элемент — тем больше воздуха вокруг (изоляция = вес, Гл. I, 6.1). Hero-заголовок и финальное обещание окружены `6xl`–`7xl`. Служебный текст — `sm`–`md`. Воздух — инструмент иерархии, а не фон.

## 2. Типографика: система

Самая важная подсистема (Гл. I, 9 — типографика это 80% дизайна).

### 2.1. КИРИЛЛИЦА — критическое требование (РФ-рынок)

Поскольку студия работает на русскоязычный рынок, **выбор гарнитуры подчинён качеству кириллицы**, а не только латиницы. Это не опция — это первый фильтр. Многие «красивые» западные premium-шрифты (Canela, Reckless, Ogg, GT Sectra и т.п.) имеют **слабую, неполную или отсутствующую кириллицу** — их нельзя брать вслепую.

**Правила выбора шрифта для AUREA на РФ-рынке:**
1. Кириллица должна быть **родной и качественной** (нарисованной типографом, а не автосгенерированной), с правильными формами я, ж, к, э, б, д, ф, щ, ъ.
2. Проверять начертания и насыщенности (если нужен variable / несколько весов — все должны иметь кириллицу).
3. Проверять кернинг кириллических пар и наличие нужных знаков (₽, кавычки-«ёлочки» « », длинное тире —, ё).

**Рекомендованные источники и гарнитуры с сильной кириллицей:**
- **Антиквы (для акцентов/заголовков):** шрифты от *type.today* и *CSTM Fonts* (напр. P't Serif-уровень для текста, но для display — Kazimir, Bandera Pro, Geometria-семейство, PT Root UI; премиальные антиквы — Cofo, Graphik-семейство с кириллицей). Из бесплатных с достойной кириллицей — **PT Serif / Pribambas, Cormorant (с кириллицей), Playfair Display (есть кириллица)**.
- **Гротески (для текста/интерфейса):** **Inter** (отличная кириллица, переменный), **Manrope**, **Onest**, **Golos Text**, **Suisse Int'l Cyrillic**, **TT-семейства** (TT Norms, TT Commons — сильная кириллица), **Graphik Cyrillic**, **Geist** (есть кириллица). Из системных-качественных — **Inter** как безопасный дефолт интерфейса.
- **Премиальные платные с кириллицей:** студии *type.today*, *CSTM Fonts*, *Brownfox* (Formular, Geometria, Pilar), *Contrast Foundry* — лучший источник «дорогих» кириллических гарнитур для РФ-рынка.

**Запрещено:** брать латинский premium-шрифт и «дотягивать» кириллицу системным фоллбэком — разнобой начертаний мгновенно читается как дёшево и ломает доверие (Гл. I, 5.2).

### 2.2. Типографическая пара

Стандарт AUREA (Гл. I, 9.1):
- **Display/Heading** — выразительная антиква **или** характерный гротеск с сильной кириллицей (характер, голос бренда).
- **Body/UI** — чистый нейтральный гротеск с отличной читаемостью кириллицы (Inter/Onest/Graphik-класс).
- Контраст пары создаёт богатство. Альтернатива — одна variable-гарнитура, контраст через вес/размер.

### 2.3. Типографическая шкала (type scale)

Коэффициент для premium-сайтов — **1.5** (драматичная) для display и **1.25** (спокойная) для текста. Базовый размер тела — 18px (для кириллицы лучше не мельчить; русский текст «плотнее» латиницы).

```
--text-xs:   13px / lh 1.5    мелкие метки, капс, обвес
--text-sm:   15px / lh 1.6    подписи, мета
--text-base: 18px / lh 1.65   основной текст (measure ≤ 65 симв.)
--text-lg:   22px / lh 1.5    крупный текст, лиды
--text-xl:   28px / lh 1.4    подзаголовки
--text-2xl:  38px / lh 1.25   заголовки секций
--text-3xl:  52px / lh 1.15   крупные заголовки
--text-4xl:  72px / lh 1.05   сцена-заголовки
--text-5xl:  104px / lh 1.0   hero-уровень
--text-6xl:  144px+ / lh 0.95 манифест, кульминации
```

Большой шаг между уровнями = смелый контраст масштабов = дорого (Гл. I, 7.1, 9.3). На мобильных шкала сжимается (раздел 9): display-размеры уменьшаются сильнее, текст — слабее.

### 2.4. Настройка (трекинг, лединг) — где рождается класс

Из Главы I (9.2):
- **Display/заголовки:** tracking от −1% до −3% (крупная кириллица «рассыпается» без отрицательного трекинга), leading 0.95–1.15.
- **Body:** tracking 0, leading 1.6–1.7 (кириллица требует чуть большего лединга, чем латиница, из-за плотности).
- **Капс-метки (kicker, мета):** tracking +6% до +14%, иначе слипается; кириллический капс особенно нуждается в разрядке.
- **Measure (длина строки):** 60–70 символов для кириллицы. Текст на всю ширину — запрещён (Гл. I, 9.2).

### 2.5. Веса и их смысл

```
Regular (400) — основной текст
Medium (500)  — лёгкий акцент, UI-элементы, лейблы
Semibold (600)— подзаголовки, кнопки
Bold (700)    — редкие сильные акценты
Light (300)   — крупные display-антиквы (тонкость = премиум)
```

Контраст весов — инструмент иерархии без смены шрифта. Не использовать «фальшивый» bold/italic (синтетический) — только реальные начертания (Гл. I, 9.5).

### 2.6. Цвет текста (не чистый чёрный)

Из Главы I (9.5): чистый `#000` на `#fff` — слишком резко, утомляет, читается «дёшево».
```
--text-primary:   #16161A  (почти чёрный, тёплый)
--text-secondary: #5A5A63  (приглушённый, для второстепенного)
--text-tertiary:  #9A9AA3  (мета, подписи)
--text-on-dark:   #F4F2EE  (почти белый на тёмном, тёплый)
```

### 2.7. Типографические запреты (свод)

Без растяжки/сжатия по ширине; не более двух гарнитур; не центрировать длинные абзацы (только короткие акценты); не оставлять «висячие» предлоги/союзы в заголовках (управлять переносами через `&nbsp;` после коротких слов); кавычки только «ёлочки» (« »), тире — длинное (—); ₽ — корректный символ; не мельчить кириллический текст ниже 15px для длинного чтения.

## 3. Вертикальный ритм (Vertical Rhythm)

### 3.1. Что это и зачем

Вертикальный ритм — согласованность вертикальных расстояний с базовой линией текста, создающая ощущение «всё выстроено». Все вертикальные отступы и высоты строк подчинены базовому ритму (кратны базовой единице/лединга). Это незаметный, но мощный источник «дорого» (Гл. I, 4.1) — глаз чувствует выверенность.

### 3.2. Правила

- межстрочный интервал body выбран так, чтобы строки ложились на ритм-сетку (напр. body 18px × 1.65 ≈ 30px ≈ кратно ритму);
- отступы до/после заголовков — кратны ритму (заголовок «дышит» больше сверху, чем снизу — он принадлежит тексту под собой: закон близости);
- блоки текста начинаются и заканчиваются на линиях ритма.

### 3.3. Когда ритм намеренно ломается

Editorial-акценты (Гл. I, 7) и broken grid (раздел 5) **сознательно** выходят из ритма — это создаёт энергию. Но: сначала строится строгий ритм, потом ломается в 1–2 местах осознанно (Гл. I, 7.3). Хаотичный сбой ритма везде = непрофессионализм.

## 4. Сетка (Grid)

### 4.1. Базовая сетка

```
Контейнер max-width: 1280–1440px (контент), full-bleed для кинокадров
Колонки (desktop): 12
Гаттер (gutter): 24–32px
Поля (margin): минимум 64px по краям на desktop, гибко на широких
```

12 колонок — гибкая основа. Но (важно) сетка — это **каркас для нарушения**, а не клетка для заполнения (Гл. I, 7.3).

### 4.2. Контейнерные ширины

- **Текстовый контейнер:** ограничен по measure (≤ 70 симв.), ~640–720px — для читаемости.
- **Контентный контейнер:** 1280–1440px — основной.
- **Широкий:** 1600px — для просторных editorial-сцен.
- **Full-bleed:** 100vw — кинокадры портфолио, full-bleed образы (Гл. I, 7.1; Гл. III, 7).

### 4.3. Асимметрия в сетке

Запрещено всё центрировать (Гл. I, 6.3). Контент смещается по правилу третей / φ: например, заголовок в колонках 1–7, образ в 8–12; или текст в 2–6, воздух справа. Асимметричное распределение колонок — стандарт, симметрия — осознанное исключение.

## 5. Broken Grid & Editorial Layout

### 5.1. Принцип

Из Главы I (7.3). Broken grid — сознательное нарушение сетки для энергии: элемент выходит за колонку, наезжает на соседний, ломает ряд, выходит за поля в full-bleed. Это противоядие «бутстрап-мышлению».

### 5.2. Приёмы editorial-вёрстки

- **Выход за колонку:** крупный образ/заголовок выходит за контентный контейнер к краю экрана.
- **Наложение (overlap):** текст наезжает на образ, образ на образ — глубина и связь (Гл. I, 12.1).
- **Крупная типографика как объект:** заголовок-графика на полэкрана (Гл. I, 9.4).
- **Editorial-обвес:** номера (01/02), рубрики-kicker, подписи-caption, folio — мелкий типографический «обвес» дорогого издания (Гл. I, 7.1).
- **Намеренная пустая колонка:** колонка оставлена пустой как пауза (форма воздуха, Гл. I, 8.2).
- **Контраст масштабов в одном кадре:** 13px рядом со 144px (Гл. I, 7.1).

### 5.3. Правила нарушения (чтобы не было каши)

1. Сначала строгая сетка → потом нарушение в 1–2 точках на сцену.
2. Каждое нарушение мотивировано (акцент, связь, переход).
3. Сохраняется якорь: невидимая линия (общий край, базовая линия, центр масс) держит композицию при сломанной сетке.

## 6. Whitespace в системе

Свод по Главе I (8) в токенах. Воздух — не «остаток», а проектируемый элемент.
- **Микро-воздух:** трекинг, лединг, padding (`3xs`–`sm`).
- **Макро-воздух:** между группами и сценами (`2xl`–`7xl`).
- **Воздух-пауза:** намеренно пустые зоны/колонки/экраны.
- Правило: сомневаешься — добавь воздуха (Гл. I, 8.3). Воздух неравномерен. Вокруг главного — максимум.

## 7. Golden Ratio в системе

Из Главы I (6.2) — камертон, не догма:
- соотношение display к подзаголовку ≈ 1.5–1.6 (см. шкалу 2.3);
- деление асимметричных композиций ≈ 62/38;
- пропорции медиа к воздуху;
- шаг шкалы (1.5 для display).
Применяется для финальной шлифовки сомнительных пропорций, не как жёсткая линейка. Смысл и баланс — первичны.
