## ГЛАВА IV. UI SYSTEM — материалы, компоненты, состояния

## 8. Цвет: система

Из Главы I (11). Сдержанная палитра: нейтральная база + один акцент. Правило 60-30-10.

### 8.1. Нейтральная база (светлая тема — основная)

```
--bg-base:      #FAF8F4   тёплый почти-белый (фон сайта, не стерильный #fff)
--bg-elevated:  #FFFFFF   приподнятые поверхности (карточки, модалки)
--bg-sunken:    #F1EEE8   утопленные зоны
--bg-dark:      #0F0F11   глубокий почти-чёрный (тёмные сцены, боль, контраст)
--bg-dark-2:    #17171A   вторичный тёмный
--border-soft:  #E7E3DB   тонкие разделители (использовать скупо!)
--border-strong:#D4CFC4   акцентные границы
```

### 8.2. Текст (из 2.6)
```
--text-primary: #16161A   --text-secondary: #5A5A63
--text-tertiary:#9A9AA3    --text-on-dark:   #F4F2EE
```

### 8.3. Акцент (один, скупо — 10%)

Акцент выбирается под эмоцию бренда (Гл. I, 11.4). Для этого проекта (мастер, доверие, премиум) — **приглушённый**, не кислотный. Варианты палитр-настроений:
```
Тёплый премиум:   --accent: #B8763E (выдержанная охра/бронза)
Строгий технологичный: --accent: #2F4A6B (приглушённый синий)
Люкс-нейтраль:    --accent: #1A1A1A (акцент = сам глубокий чёрный/контраст)
```
Акцент работает потому, что его мало (Гл. I, 11.1). Запрещены чистые цвета (#FF0000) и диско-градиенты (Гл. I, 11.2).

### 8.4. Тёмная тема как драматургический инструмент

Тёмные сцены (боль, кульминации портфолио) — не «инверсия», а **отдельная атмосфера**. Тёмный фон с зерном (8.7), тёплый почти-белый текст, акцент света. Переключение свет↔тьма по ходу сайта = смена освещения сцены (Гл. II, 7.4).

### 8.5. Семантика состояний (приглушённая)
```
--success: #4A7C59 (приглушённый зелёный)
--error:   #A5453B (приглушённый терракот, не #FF0000)
--warning: #B8862F
```
Даже служебные цвета — выдержанные, не кричащие (Гл. I, 11.3).

## 9. Материалы и фактура (система)

Из Главы I (10). Достоверность — ключ. Бутафория = дёшево.

### 9.1. Стекло (glass)
```
background: rgba(255,255,255,0.06–0.12);
backdrop-filter: blur(16–24px) saturate(1.2);
border: 1px solid rgba(255,255,255,0.10);
+ тонкая верхняя подсветка края (inset highlight)
```
**Где:** плавающая навигация, оверлеи, модалки. **Где нельзя:** как фон всего подряд, поверх текста (читаемость), на каждом элементе. Тонко: лёгкое размытие, аккуратная грань. Грубое стекло (сильный blur, толстая белая обводка, кислота под ним) = дёшево (Гл. I, 10.1).

### 9.2. Камень / бумага / матовость
Тёплая матовая фактура через едва различимый шум + тональность. Язык люкса (беж, тёплый серый). Антипод глянцу.

### 9.3. Металл
Тонкий градиентный блик на тексте/линиях (полированный металл) для логотипа/акцента. Light Sweep (Гл. II, 7.4). Доза минимальна — передоз = «казино» (Гл. I, 10.1).

### 9.4. Текстуры
Едва заметные: тканевая/бумажная фактура фона, тонкая сетка-grid как фон editorial-сцены. Всегда подложкой, никогда не конкурируют с контентом.

## 10. Фон, шум, градиенты

### 10.1. Фон проектируется первым (не «остаётся белым»)
Из Главы I (10.3). Фон задаёт тональность. Варианты: тёплый почти-белый (#FAF8F4), глубокий почти-чёрный с зерном (#0F0F11), мягкий тональный градиент в пределах одного оттенка.

### 10.2. Шум (grain) — обязательный ингредиент
Из Главы I (10.2). Тонкое зерно поверх заливок/градиентов: 3–8% opacity, мелкое. Убирает бандинг, добавляет «аналоговости», глубины. Признак дорогого сайта. Реализация — SVG-шум или PNG-текстура, `mix-blend-mode: overlay/soft-light`, `pointer-events: none`.

### 10.3. Градиенты — только сложные и тональные
**Запрещены** диско-градиенты по умолчанию (фиолетово-розовый и т.п., Гл. I, 11.2). **Разрешены:**
- тональные (оттенки одного цвета): мягкий переход внутри нейтрали;
- свет-градиенты (имитация источника света): радиальный, едва заметный, от тёплого блика к фону;
- mesh-градиенты приглушённых тонов **с шумом** поверх (без шума mesh выглядит «стоково»).
Градиент должен ощущаться как свет/атмосфера, а не как «заливка кнопки».

## 11. Освещение и глубина (система)

Из Главы I (12). Один источник света на сцену, единое направление. Глубина — тонкая, послойная.

### 11.1. Слои (z-уровни)
```
z-base:     фон (с шумом)
z-content:  основной контент
z-raised:   карточки/приподнятое (мягкая тень)
z-overlay:  стекло/модалки/навигация (blur)
z-cursor:   кастомный курсор, поверх всего
```
Слои разделяются тонко: размытие дальних, разная скорость при скролле (параллакс, Гл. II, 9), масштаб, лёгкое затемнение — не толстыми тенями (Гл. I, 12.1).

## 12. Тени: система (КРИТИЧНО)

Из Главы I (12.3). **Запрещена единая тень на всех элементах.** Тень = функция высоты над фоном. Многослойные, мягкие, окрашенные тени.

### 12.1. Токены теней (светлая тема, тёплая окраска)
```
--shadow-xs:  0 1px 2px rgba(30,25,20,0.04);
--shadow-sm:  0 2px 4px rgba(30,25,20,0.05),
              0 1px 2px rgba(30,25,20,0.04);
--shadow-md:  0 4px 12px rgba(30,25,20,0.06),
              0 2px 4px rgba(30,25,20,0.05);
--shadow-lg:  0 12px 32px rgba(30,25,20,0.08),
              0 4px 8px rgba(30,25,20,0.05);
--shadow-xl:  0 24px 64px rgba(30,25,20,0.10),
              0 8px 16px rgba(30,25,20,0.06);
--shadow-glow:0 0 48px rgba(184,118,62,0.15); /* свет акцента */
```

### 12.2. Правила теней
- тень **окрашена** в тон фона/объекта (тёплая на тёплом), не чистый чёрный (Гл. I, 12.2);
- **многослойная** (2+ тени разного радиуса) = реалистичность вместо плоского штампа;
- чем «выше»/важнее элемент — тем больше радиус и мягкость;
- кнопка, карточка, модалка, навигация-на-скролле — **разные** тени;
- на тёмной теме тени почти не работают — глубину даёт свет (свечение, осветление края), а не затемнение.

## 13. Blur

- **Backdrop-blur:** стекло (9.1), оверлеи, навигация на скролле.
- **Layer-blur:** дальние слои глубины (Гл. I, 12.1), расфокус неактивного (Гл. II, 7.2).
- **Motion-blur (имитация):** на быстрых переходах для кинематографичности (дозированно).
- Запрещён blur поверх читаемого текста; blur ради blur.

## 14. Контраст

### 14.1. Контраст как иерархия
Контраст (размера, веса, цвета, пустоты) — главный инструмент иерархии (Гл. I, 6.1). Premium-дизайн строится на **сильных контрастах масштаба** (13px ↔ 144px) при **сдержанных контрастах цвета** (нейтраль + один акцент).

### 14.2. Контраст доступности
Текст должен соответствовать WCAG AA (≥4.5:1 для body, ≥3:1 для крупного). Почти-чёрный на почти-белом (#16161A на #FAF8F4) проходит. Приглушённый вторичный текст — проверять, не уходить ниже нормы ради «эстетики бледности» (бледный нечитаемый текст = и недоступно, и непрофессионально).

## 15. Карточки: когда можно, когда КАТЕГОРИЧЕСКИ нельзя

Это один из важнейших разделов — карточки главный источник «дешёвого блочного» вида (Гл. I, 5.2; Гл. III, 5.1).

### 15.1. Когда карточки ЗАПРЕЩЕНЫ
- для **ценностей/преимуществ** («что получаете») — это editorial-перечень, не карточки (Гл. III, 5);
- для **портфолио** — это full-bleed кинокадры, не карточки (Гл. III, 7; прямое требование контента «не карточки»);
- для **отзывов** — это цитаты-развороты (Гл. III, 9);
- для **этапов работы** — это sticky/линия-путь (Гл. III, 4);
- для **тарифов** — это прайс-манифест, не колонки-карточки (Гл. III, 6);
- **ВСЕГДА запрещены:** ряды одинаковых карточек с иконкой-в-кружочке + заголовком + текстом (это «бутстрап-мышление», смерть индивидуальности).

### 15.2. Когда карточки ДОПУСТИМЫ
Карточка оправдана, только если контент **по своей природе дискретен, равнозначен и требует параллельного сравнения**, и нет лучшего editorial-решения. Редкие случаи:
- интерактивные элементы выбора (если правда нужны);
- вторичные служебные списки.
Даже тогда — карточки должны быть **разными** (размер, акцент, broken grid), а не клонами в ряд.

### 15.3. Если карточка всё же нужна — правила
- разный размер/вес (иерархия, не клоны);
- broken-grid расположение, не ровный ряд 3×N;
- тень по высоте (12), не единая;
- щедрый внутренний воздух;
- осмысленный hover (16);
- без обводки-1px-вокруг-всего (Гл. I, 5.2).

> Правило студии: прежде чем сделать карточку — докажи, почему это не editorial-решение. По умолчанию — НЕ карточка.

## 16. Кнопки

### 16.1. Иерархия кнопок
- **Primary (одна на экран):** заполненная, акцент/тёмная, главное действие («Обсудить проект»).
- **Secondary:** контурная или текстовая («Смотреть работы»).
- **Tertiary/ghost:** текст со стрелкой/подчёркиванием.
Одна primary на сцену (один фокус, Гл. I, 6.1).

### 16.2. Размеры и токены
```
Высота: sm 40px / md 48px / lg 56px
Padding гориз.: 20–32px
Радиус: --radius-btn (см. 19); часто 8–12px или pill (999px) для премиум
Шрифт: 15–16px, weight 500–600, tracking +1%
```

### 16.3. Состояния (ВСЕ обязательны)
```
default → hover → active(pressed) → focus-visible → loading → disabled → success
```
- **hover:** НЕ просто смена цвета (запрет, Гл. II, 12.3). Премиальный hover: лёгкий подъём + усиление тени, или заливка «прорисовывается», или блик проходит (Light Sweep), или магнитность (Гл. II, 12.2). 150–250ms, expo.
- **active:** лёгкое «вдавливание» (scale 0.98), мгновенно.
- **focus-visible:** заметное кольцо для клавиатуры (доступность), не убирать.
- **loading:** спиннер/прогресс, не «мёртвая» кнопка.
- **success:** микроанимация подтверждения (галочка morph).
- **disabled:** приглушённая, ясно нерабочая.

### 16.4. Магнитные primary-кнопки
Главные CTA — магнитные (притяжение к курсору ≤15px, мягкая пружина, Гл. II, 12.2). Тактильность = премиум (Гл. I, 3.1).

## 17. Формы и поля

Критично для финала (Гл. III, 11) — форма это кульминация действия.

### 17.1. Принцип «форма-разговор»
Поля — не казённые боксы, а человеческий диалог (Гл. III, 11.4). Минимум полей. Лейблы человеческие.

### 17.2. Поля ввода: токены и состояния
```
Высота: 52–56px (комфорт касания, особенно мобайл)
Padding: 16–20px
Радиус: согласован с кнопками
Border: 1px --border-strong; фон --bg-elevated или прозрачный
Шрифт: 16px+ (на мобайле <16px вызывает зум iOS — запрещено)
```
Состояния: `default → focus (плавная подсветка границы/свечение акцента) → filled → error (мягкий терракот + человеческая подсказка, не «ОШИБКА!») → success → disabled`.

### 17.3. Микровзаимодействия формы
- focus: мягкое появление подсветки границы (200ms), не резкий outline;
- ввод: плавный floating-label или плавная смена состояния;
- валидация: мягкая, человечная, по месту — не агрессивный красный;
- отправка: loading → достойный success-экран (Гл. III, 11.8), не унылый alert.

### 17.4. РФ-специфика форм
- кнопки связи через **мессенджеры** (Telegram, WhatsApp) наряду с формой — РФ-аудитория часто предпочитает написать в Telegram, а не оставлять email (см. финальный ответ);
- телефон с маской +7;
- честность про обработку данных (152-ФЗ): тихая ссылка на согласие, без навязчивости.

## 18. Иконки

- **Стиль:** единый набор, тонкая линия (stroke 1.5–2px) или аккуратный солид — но не микс. Геометрия согласована.
- **Запрещено:** стоковые цветные иконки в кружочках (Гл. I, 5.2), эмодзи как иконки (Гл. I, 5.2), микс стилей.
- **Размеры:** 16 / 20 / 24 / 32 (кратно 4).
- **Анимация:** SVG Draw при появлении (Гл. II, 13.3), morph между состояниями (Гл. II, 13.2).
- Иконки — поддержка, не герой. Не заменяют типографику в роли носителя смысла.

## 19. Радиусы

```
--radius-xs:  4px   мелкие элементы, теги
--radius-sm:  8px   поля, мелкие кнопки
--radius-md:  12px  кнопки, карточки (если есть)
--radius-lg:  20px  крупные поверхности
--radius-xl:  32px  модалки, крупные блоки
--radius-pill:999px кнопки-pill, теги
--radius-none:0     editorial-кадры, full-bleed, строгие сцены
```
Радиус — часть характера: острые углы (0) = строго/редакционно; pill = мягко/дружелюбно. **Один язык радиусов на проект** — не миксовать случайно. Запрет: разные радиусы на однотипных элементах.

## 20. Мобильная адаптация (критично для РФ-рынка)

РФ-трафик преимущественно мобильный, часто на средних/слабых устройствах. Адаптив — не «сжать десктоп», а **переосмыслить** (Гл. VI: своя композиция).

### 20.1. Брейкпоинты
```
mobile:  ≤ 600px   (основной трафик РФ)
tablet:  601–1024px
desktop: 1025–1440px
wide:    ≥ 1441px
```

### 20.2. Правила мобайла
- **Type scale сжимается:** display-уровни падают сильнее (5xl/6xl → ~32–44px), body почти не меняется (16–18px, не мельче).
- **Воздух сохраняется пропорционально** — не уплотнять в кашу ради «уместить»; премиальность = воздух и на мобайле.
- **Кастомный курсор отключается** (нет мыши); магнитность → тап-отклик.
- **Тяжёлые приёмы упрощаются:** сложный параллакс/3D/частицы — облегчённые версии или отключение (производительность, Гл. II, 4.4); 60fps на среднем Android — обязательно.
- **Sticky storytelling** — пересматривается под тач (часто короче/проще).
- **Тач-цели** ≥ 44×44px; поля ≥16px шрифт (анти-зум iOS).
- **`prefers-reduced-motion`** уважается (Гл. II, 5.3).
- Композиции, ломающие сетку на десктопе, **перекомпоновываются** под вертикаль — не просто стакаются.

## 21. Сводные запреты главы (UI)

1. Запрещено единообразие из токенов — система это палитра, не раскладка (0).
2. Запрещены одинаковые вертикальные отступы между сценами (1.3).
3. Запрещён латинский premium-шрифт с фоллбэк-кириллицей (2.1).
4. Запрещено центрировать длинные тексты и всю композицию (2.7; 4.3).
5. Запрещены чистые цвета и диско-градиенты (8.3; 10.3).
6. Запрещён фон «по умолчанию белый» без шума и тональности (10).
7. Запрещена единая тень на всех элементах (12).
8. Запрещены ряды одинаковых карточек; по умолчанию — НЕ карточка (15).
9. Запрещён hover «только смена цвета»; обязательны все состояния (16.3).
10. Запрещены стоковые иконки в кружочках и эмодзи-иконки (18).
11. Запрещён микс стилей иконок/радиусов (18, 19).
12. Запрещён мобайл как «сжатый десктоп»; шрифт полей <16px (20).
13. Запрещён контраст ниже WCAG AA ради «бледной эстетики» (14.2).

## 22. Чек-лист UI перед сдачей сцены

1. Шрифт имеет качественную родную кириллицу? Пара настроена (трекинг/лединг)?
2. Отступы по шкале, но неравномерные (иерархия, не механика)?
3. Вертикальный ритм выдержан (и осознанно нарушен в 1–2 точках)?
4. Сетка асимметрична? Есть broken-grid акцент? Есть якорь композиции?
5. Воздух пропорционален важности? Вокруг главного — максимум?
6. Фон спроектирован (тональность + шум)? Не стерильный белый?
7. Палитра сдержанна (нейтраль + один акцент, 60-30-10)?
8. Тени по высоте, многослойные, окрашенные — НЕ единая?
9. Нет карточек-сеток? Каждое «карточное» решение оправдано как не-editorial?
10. Кнопки: одна primary, все состояния, премиальный hover, магнитность CTA?
11. Формы: «разговор», микровзаимодействия, ≥16px поля, мессенджеры (РФ)?
12. Иконки единого стиля, без кружочков/эмодзи? Радиусы — один язык?
13. Мобайл переосмыслен (не сжат)? 60fps? Тач-цели? reduced-motion?
14. Контраст проходит WCAG AA?
15. Финальный тест (Гл. I, 5.3): тест бутика / тишины / одного слова пройден?
